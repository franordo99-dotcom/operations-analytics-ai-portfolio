# Privacidad de Datos

English version: [data-privacy.md](data-privacy.md)

Este portfolio está diseñado con privacidad como prioridad.

Los case studies están escritos para revisión pública, por lo que evitan publicar información privada, confidencial u operativamente sensible. Algunos casos pueden estar basados en workflows reales, pero fueron adaptados y sanitizados antes de su publicación.

## Qué no se publica

Este repositorio no publica:

- datos reales de empresa;
- datos de clientes o proveedores;
- información de empleados;
- facturas reales o documentos comerciales reales;
- importes reales o condiciones comerciales;
- identificadores de registros ERP;
- API keys, tokens, passwords o credenciales;
- logs internos, capturas, bases de datos o planillas;
- rutas privadas, paths locales o artefactos operativos.

## Datos demo

Los archivos demo son sintéticos. Se incluyen solo para ilustrar la forma de un workflow, como un payload parseado, un registro de staging o un resultado de auditoría.

Los datos demo sintéticos no deben interpretarse como evidencia de producción, performance, precisión o resultados de negocio.

## Principio de publicación

La versión pública de cada case study está diseñada para mostrar razonamiento profesional mientras protege el contexto privado. El foco está en el problema de negocio, el diseño del proceso, los puntos de control y los aprendizajes.
